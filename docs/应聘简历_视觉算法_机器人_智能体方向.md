### 多模块 AI 应用系统

**项目名称：** Pediatrics RAG + Vision + Detection + Robotics + macOS App  
**项目角色：** 独立完成方案设计、开发实现、联调验证与文档沉淀  
**技术栈：** `Python`、`PyTorch`、`OpenCV`、`FastAPI`、`SwiftUI`、`SQLite`、`YOLOv8n`、`ResNet18`

**项目简介：**  
围绕“知识问答 + 视觉识别 + 机器人执行演示”的落地方向，独立搭建一个多模块 AI 项目，形成从模型能力、推理服务到桌面端交互展示的完整闭环。项目当前已拆分为 `RAG / Vision / Detection / Robotics` 四个模块，便于演示、扩展和后续工程化迭代。

**项目职责与亮点：**

- 设计并实现本地 `RAG` 知识问答链路，支持 `PDF / DOCX / 图片 / TXT / Markdown / PPTX / XLSX / HTML / CSV / JSON / XML / EPUB` 等多类资料接入。
- 为文档转换、清洗、切块和向量化建立缓存与增量复用机制，支持知识库自动检测源文件变化并重建索引。
- 使用 `FastAPI` 封装统一本地服务，打通知识问答、视觉推理、检测结果和机器人决策数据输出。
- 使用 `SwiftUI` 开发 `macOS` 原生客户端，客户端启动时自动拉起本地 Python 服务并做健康检查，形成开箱可演示的桌面应用。
- 将训练标注数据管理从平面文件升级为 `SQLite` 主存储，支持训练样本维护、上下文刷新、导出快照和生成 `SFT` 数据集。
- 规划并接入 `LoRA / QLoRA` 训练脚本，明确“`RAG` 负责外部知识检索，`LoRA` 负责回答行为对齐”的工程边界。

### 视觉分类与开放场景拒识实验

**项目目标：**  
基于商品图完成 `diaper / stroller` 二分类实验，并在推理阶段支持 `other` 拒识输出，验证小规模视觉分类闭环和开放场景拒识方案。

**职责与成果：**

- 使用 `ResNet18` 完成训练、验证、测试和单图推理闭环。
- 设计并实现两套拒识方案：`softmax` 置信度阈值拒识、`prototype` 原型相似度拒识。
- 默认采用 `prototype` 拒识方案，提升“已知类分类 + 未知类拒识”场景下的可解释性。
- 当前测试集指标达到：`accuracy 0.9787`、`macro F1 0.9787`，最佳验证集准确率 `0.9091`。
- 支持原型构建、批量验证、手工检查样例输出和推理结果解释，形成完整实验流程。

### 目标检测与实时视觉推理闭环

**项目目标：**  
搭建 `diaper / stroller / phone` 三类目标检测最小闭环，覆盖数据组织、标注、训练、评测、单图推理和摄像头实时演示。

**职责与成果：**

- 基于 `YOLOv8n` 搭建检测数据集、训练脚本、评测脚本和单图推理可视化链路。
- 组织并维护检测数据集，当前数据规模为：`train 93`、`val 10`、`test 7`，另有 `rejected 7` 张排除样本。
- 完成第一轮检测训练与评测，测试集达到：`mAP50 = 0.601`、`mAP50-95 = 0.374`。
- 输出分类别结果，其中 `diaper mAP50 = 0.525`、`stroller mAP50 = 0.677`。
- 基于 `OpenCV` 封装摄像头实时检测 demo，支持读帧、预处理、周期性推理、结果叠加和 FPS 展示。
- 在桌面端支持运行时调节检测阈值，便于现场演示、误检漏检分析和阈值对比验证。

### 机器人视觉流程演示模块

**项目目标：**  
在不虚构真实机械臂控制能力的前提下，把视觉检测结果转为机器人任务阶段、抓取点和分拣决策，构建适合机器人软件岗位展示的应用层 demo。

**职责与成果：**

- 独立设计 `robotics/` 模块，明确与 `vision/`、`detection/` 的边界，专注“视觉识别 -> 任务决策 -> 动作演示”的应用层表达。
- 复用目标检测结果，构建 `Detect -> Target Lock -> Path Plan -> Pick -> Place` 五阶段任务时间线。
- 将检测结果转换为 `target`、`pick_point`、`destination_bin`、`selection_reason` 等结构化决策输出。
- 默认采用最高置信度目标作为任务目标，并以 `bbox` 中心点生成抓取点，形成最小可运行抓取演示逻辑。
- 在 `macOS` 客户端中实现 `Robotics / Playground` 与 `Robotics / Workflow` 两个页面，展示视觉链路、运行快照、阶段推进和分拣逻辑解释。
- 技术表达清晰聚焦 `OpenCV -> PyTorch -> Industrial Vision Logic -> Robot Action`，便于对接机器人视觉、工业视觉软件研发岗位。
